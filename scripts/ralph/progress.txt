## Codebase Patterns
- Keep the root `AGENTS.md` authoritative for repository rules; fixture-level `AGENTS.md` and `CLAUDE.md` files under `tests/fixtures/` are scanner inputs, not active project instructions.
- Scanner modules return `Finding` objects with `id`, `title`, `severity`, `scanner`, `why`, and `next`; include `file`, `line`, and redacted `evidence` when applicable.
- Report formats are `text`, `markdown`, `json`, `html`, `sarif`, and `annotations`; `writeReports` writes file-backed formats and skips console-only `text`/`annotations`.
- Generated local outputs are intentionally ignored via `.gitignore`: `dist/`, `.tmp/`, `.agent-reliability/`, logs, root Codex `progress*.md` files, and real `.env` files.
- Do not commit Ralph local run traces such as `scripts/ralph/planner.last.txt`, `.last-branch`, or `archive/`; they may contain machine-local paths or transient logs.
- CLI behavior tests should call the exported `runCli(argv, io)` helper with injected stdout/stderr/cwd instead of spawning the compiled `dist/cli.js`; `npm test` runs before `npm run build` in the full check script.
- Release readiness is now a first-class scanner in `src/scanners/releaseReadiness.ts`; keep README trust checks in `readme.ts` and package/output-contract checks in the release scanner.
